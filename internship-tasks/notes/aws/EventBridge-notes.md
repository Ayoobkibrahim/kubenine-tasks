* Amazon EventBridge — Core Concepts

1. What EventBridge is and how it differs from SNS and SQS

EventBridge is a serverless event bus that routes data between applications and AWS services based on rules you define. While SQS simply holds messages in a queue and SNS broadcasts messages to subscribers, EventBridge acts as a smart router that filters and choreographs complex events across your entire cloud architecture.

2. What an event bus is — the default bus vs custom event buses

An event bus is the central pipeline that receives and distributes events within your account. The "default bus" automatically receives operational events from AWS services, while "custom buses" are created by you to handle specific business events generated by your own applications.

3. What a rule is

A rule acts as a routing filter attached to an event bus that constantly evaluates incoming events. If an event matches the rule's exact criteria, the rule captures it and securely forwards it to one or more designated target services.

4. Two types of rules: scheduled rules and event pattern rules

Scheduled rules act like traditional cron jobs, triggering actions at specific times or intervals (e.g., "run this billing script every Monday"). Event pattern rules act like conditional filters, triggering actions only when the actual JSON payload of an incoming event matches your specified parameters.

5. How AWS services automatically publish events to the default event bus

AWS services are deeply integrated to automatically emit state changes directly to your account's default event bus. For example, if an EC2 instance stops running or someone alters an S3 bucket configuration, AWS instantly fires a JSON event detailing exactly what happened onto the default bus.

6. How event patterns work

Event patterns use a JSON format to create a strict matching template for incoming data. If the structure and specific key-value pairs in your pattern match the incoming event's JSON payload, EventBridge considers it a match and fires the rule.

7. What targets EventBridge can route to

Once a rule matches an event, EventBridge can push that event payload to a wide variety of AWS targets. Common targets include triggering a Lambda function to run code, starting a Step Functions state machine, or dropping the event into an SNS topic or SQS queue for downstream processing.
