## What happens when a target goes DOWN

When a target goes down, Prometheus **stops scraping** new metrics from that endpoint. **Existing metric samples remain** in the TSDB for a while but **eventually become stale** because no new data is received. This affects dashboards, monitoring visibility, and alerting accuracy.

## PromQL expression to detect DOWN targets

```promql
up == 0
```

Use this in the **Graph** tab to list series where scraping failed, or in alert rules to notify when any target is unreachable. You may narrow it with label matchers, for example `{job="node-exporter"}`.

## Explanation of the `up` metric

The **`up`** metric is automatically generated by Prometheus **for every scrape target**.

| Value | Meaning |
|-------|--------|
| **`up = 1`** | Target is healthy (last scrape succeeded). |
| **`up = 0`** | Target is DOWN or unreachable (last scrape failed). |

This metric is commonly used in Prometheus alert rules to detect monitoring failures.
